Add a fourth standing-orders role card to the Flows page (#123, epic
#122): "Claude Code — Product Architect". The card establishes the
direction/architecture role — decides and documents, never implements,
never dispatches chips, never merges; output is issues, ADRs, and specs
the orchestrator picks up. It points at the role's toolset (/history
recall, DECISIONS.md grep, board archaeology via gh, as-built read-only
code survey vs as-intended VISION/specs, research under the honesty bar)
and its guardrails (options-with-costs at forks, evidence class on every
claim, epic fan-out stops at the board). The Standing orders heading now
carries the menu rule — pick the one card matching the session's role,
never stack cards — rendered under the heading in both the prompt
library and the map's node drawer; PromptList now owns a group's intro,
items, and note in one place instead of both surfaces duplicating them.
