# /adr skill: decision capture + DECISIONS.md backfill

New user-level skill `skills/adr/SKILL.md` (#124, epic #122): capture an
architecture/product decision as a dated ADR — options considered with
costs, the decision, consequences, and evidence citations (issue #s,
commit shas, PR #s, dated prior entries — "why with receipts, not just
what", the pattern surfaced by the anduin#208 survey). Entries land twice,
**issue-first**: mirrored as a comment on the owning issue, then appended
newest-at-top to `docs/DECISIONS.md` on the owning branch. The skill never
loads the whole file to append — it reads only the header and the first
entry's heading, per the canon's shared-files rules — and creates the file
with a two-line header when absent. Stack-neutral; reads only owner/repo
from the repo's `GITHUB-PROJECTS.md` config block.

`/update-docs` routing: the decision-tree "Design decision" row now routes
to `/adr` for the actual capture (one-cell edit).

**Live demonstration in the same PR:** `docs/DECISIONS.md` created via the
skill's own procedure, backfilling the epic #132 fork record as its first
entry — Forks A2 (route rename + permanent redirect), B2 (shared FlowMap),
C2 (cardinality on home + Architect aside), and the serial #134 → #133 →
#135 sequencing amendment, operator-approved 2026-07-18 — mirrored as a
comment on #132. The registry listing follows automatically (skills/ is
the source of truth; served copies regenerated via sync:skills).
